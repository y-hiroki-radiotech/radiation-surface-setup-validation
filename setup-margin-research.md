# 放射線治療におけるセットアップマージンの考え方

## 目的

放射線治療のセットアップマージン内に標的が収まっているかどうかを判定するコードを作る前提として、まずマージンの考え方と実装上の前提を整理する。

## 結論の要点

セットアップマージンは、単純に標的の外側へ一定距離を足す概念ではない。標準的には、標的の内部運動とセットアップ誤差を分けて扱う。

- `CTV + Internal Margin = ITV`
- `ITV + Setup Margin = PTV`

ICRU系の考え方では、`PTV` は幾何学的不確実性を見込んで、治療期間を通して `CTV` に意図した線量を与えるための幾何学的概念である。

## 基本用語

### CTV

Clinical Target Volume。肉眼病変や微小進展を含めて治療すべき標的。

### Internal Margin / ITV

呼吸や臓器運動など、患者内部の変動を見込むためのマージン。  
`ITV` は内部運動を考慮した標的。

### Setup Margin

患者セッティング誤差、固定具の再現性、画像照合後の残差誤差、装置の幾何学精度などを見込むためのマージン。

### PTV

Planning Target Volume。`CTV` または `ITV` に対して、幾何学的不確実性を補償するために設定する計画用体積。

## セットアップマージンの考え方

セットアップマージンは、部位ごとに固定値を使うものではなく、少なくとも次の要素に依存する。

- 解剖学的部位
- 固定具と体位
- 画像誘導の方法
- 骨照合か軟部組織照合か
- オンライン補正の有無
- 分割回数
- intrafraction motion の大きさ
- 施設固有の運用精度

そのため、文献値をそのまま採用するのではなく、施設ごとの IGRT 実測データから決めるのが望ましい。

## 系統誤差と偶然誤差

セットアップマージンの議論では、誤差を次の 2 種類に分ける。

- `Σ` : systematic error
- `σ` : random error

### `Σ` 系統誤差

患者ごとに治療期間を通して持続する偏り。  
例:

- シミュレーション時と治療時の再現性のずれ
- 固定具の一貫したズレ
- 参照座標系の偏り

### `σ` 偶然誤差

各 fraction ごとに変動するばらつき。  
例:

- 毎回のわずかな位置ずれ
- 日々の姿勢差
- 照合後の残差誤差

## 代表的なマージンレシピ

### Stroom

`M = 2Σ + 0.7σ`

### van Herk

`M = 2.5Σ + 0.7σ`

van Herk の式は、一般に「90% の患者で CTV 最小線量が処方線量の 95% 以上になる」ことを狙うレシピとして広く使われる。

## 実装上の重要点

### 1. 計画時のマージン設定と、日々の判定は別問題

コードとしては次の 2 つを分ける必要がある。

- `計画時`: 過去データから `Σ` と `σ` を計算してマージンを決める
- `治療当日`: 当日の位置ずれや輪郭情報を使って、標的が計画上の許容範囲に入っているか判定する

### 2. 方向別異方性マージンを前提にした方がよい

実臨床では LR/AP/SI で誤差特性が異なることが多い。  
そのため、最初の実装から等方性 1 値よりも、方向別マージンを扱える設計が望ましい。

例:

- `margin_lr`
- `margin_ap`
- `margin_si`

### 3. 最初の判定器は残差シフトベースで作れる

3D 輪郭がなくても、当日の残差シフトが方向別マージン内かどうかで暫定判定は可能。

判定例:

- `abs(shift_lr) <= margin_lr`
- `abs(shift_ap) <= margin_ap`
- `abs(shift_si) <= margin_si`

すべて満たせば、まずは「マージン内」と判定する。

### 4. ただし残差シフトだけでは不十分

次の問題は残差シフトのみでは評価しきれない。

- 回転誤差
- 標的形状変化
- 軟部組織の変形
- 骨照合と実標的位置の乖離
- intrafraction motion

そのため、最終的には `当日の CTV/ITV が計画 PTV に 3D で包含されるか` を見る設計に拡張できるようにしておくべきである。

### 5. 骨照合と軟部照合は分けて考える

骨性ランドマークは位置合わせの代理指標として有用だが、軟部標的の真の位置と一致するとは限らない。  
特に胸腹部や骨盤部では、骨照合だけで標的包含を保証できない場合がある。

### 6. IGRT / Adaptive RT ではマージン縮小があり得る

daily CBCT、SGRT、online adaptation などを導入すると残差誤差は小さくなり、マージン縮小は理論上可能である。  
ただし、その縮小は各施設のワークフローで検証されたデータに基づく必要がある。

## IDENTIFY を使った体表面治療向け判定方針

### 前提

体表面位置照合装置 `IDENTIFY` で次の情報が見えている。

- `Vertical`
- `Long`
- `Lateral`
- `Mag`
- `Rotation`
- `Pitch`
- `Roll`

この情報を使って、標的がセットアップマージン内に収まっているかを判定したい。  
想定ユースケースは、主として体表面治療、乳房、胸壁など、体表面情報と標的位置の相関が比較的高い領域である。

ただし重要なのは、`体表面 = 真の標的位置` ではないことである。  
SGRT は有用な代理指標だが、部位、ROI の選び方、表面形状、呼吸や姿勢変化により、標的位置との乖離が生じる。

## IDENTIFY の値をどう使うべきか

`Mag` は並進成分の大きさをまとめた表示としては有用だが、判定の主指標としては不十分である。  
理由は次の通り。

- 方向情報を失う
- 回転の影響を含められない
- ISO から遠い点ほど危険、という性質を表現できない

そのため、判定は `Vertical / Long / Lateral / Rotation / Pitch / Roll` の 6DoF を使い、`ISOセンターから遠い評価点での実効変位` に変換して行うのが妥当である。

## 基本となる幾何学モデル

IDENTIFY の観測値を、剛体変換の

- 並進 `t`
- 回転 `R`

として扱う。

ISO 基準座標系における評価点 `p_i` に対し、その点での位置ずれは概念的に次で表せる。

`Δp_i = t + (R - I) p_i`

ここで

- `t` は並進ベクトル
- `R` は回転行列
- `I` は単位行列
- `p_i` は ISO から見た評価点の位置ベクトル

である。

この式の意味は、`同じ回転角でも ISO から遠い点ほど大きく動く` ということである。

## 遠位点の考え方

今回の主目的は、`ISO センターから最も遠い部分、またはある程度遠い部分がセットアップマージンから外れないか` を判定することにある。  
したがって、評価点は少なくとも次のどれかで持つべきである。

### v1 の評価点

- 手入力した代表点
- 最遠点 1 点
- 遠位点群 3 点から 10 点程度

### v2 以降

- CTV または関心領域輪郭上の表面点群
- `ISO からの距離が上位 x%` の点群
- DICOM RTSTRUCT から自動抽出した点群

単一の最遠点だけでは局所的な例外に弱いため、実務上は `最遠点` と `ある程度遠い点群` を両方見るのが望ましい。

## 回転による遠位点変位

小さい回転角では、回転による変位の大きさは概ね

`回転変位 ≈ 距離 × 角度(rad)`

で近似できる。

例:

- `1° = 0.01745 rad`
- `ISO から 100 mm` 離れた点では `約 1.7 mm`
- `ISO から 150 mm` 離れた点では `約 2.6 mm`

つまり、見かけ上の回転が小さくても、遠位点では無視できないことがある。

## マージン判定の中心指標

最初に実装すべき中心指標は、`最悪点の実効変位` である。

各評価点ごとに少なくとも次を算出する。

- `translation_only_mm`
- `rotation_induced_mm`
- `effective_displacement_mm`
- `distance_from_iso_mm`

そのうえで、全評価点のうち最大のものを `worst_point` として採用する。

## 不確かさの扱い

IDENTIFY 自体にも不確かさがあるため、観測値だけで閾値判定するのではなく、不確かさを保守的に上乗せして判定する。

少なくとも次の不確かさを考慮する。

- `U_identify`
  - 装置自体の測定不確かさ
- `U_surrogate`
  - 体表面と真の標的位置の乖離
- `U_registration`
  - ROI 設定や参照面設定の不確かさ
- `U_intrafraction`
  - 治療中の体動や呼吸変動
- `U_model`
  - 幾何モデル簡略化による不確かさ

初期実装では、これらを独立とみなして二乗和平方根で合成する。

`U_total = sqrt(U_identify^2 + U_surrogate^2 + U_registration^2 + U_intrafraction^2 + U_model^2)`

安全側に倒した判定では、例えば 95% 相当の係数 `z = 1.96` または簡便に `z = 2.0` を使い、

`保守的変位 = 実効変位 + z × U_total`

とする。

## 軸別マージン判定

実臨床では、LR/AP/SI または Vertical/Long/Lateral ごとに許容値が異なることがある。  
そのため、判定は等方性 1 値よりも方向別に持てる設計が望ましい。

点 `i` の軸 `k` に対して、判定の基本形は次の通り。

`abs(Δp_i,k) + z × U_i,k <= M_k`

ここで

- `Δp_i,k` : 点 `i` の軸 `k` における変位
- `U_i,k` : 点 `i` の軸 `k` における総合不確かさ
- `M_k` : 軸 `k` のマージン

である。

また、表示用には等価的な 3D 指標も併記できる。

- `3D displacement`
- `3D conservative displacement`
- `margin consumption ratio`

## 推奨する判定結果

判定結果として少なくとも次を返すとよい。

- `pass_fail`
- `worst_point_name`
- `distance_from_iso_mm`
- `translation_only_mm`
- `rotation_induced_mm`
- `effective_displacement_mm`
- `uncertainty_mm`
- `conservative_displacement_mm`
- `margin_remaining_mm`
- `axiswise_status`

## 臨床運用上の注意

体表面治療では SGRT は非常に有用だが、深部標的や変形の大きい部位では、表面ずれと標的ずれが一致しない可能性がある。  
したがって、`U_surrogate` は最重要パラメータのひとつであり、可能なら過去の CBCT や kV 画像との比較から施設固有値として設定するべきである。

また、SGRT は ROI の選び方に依存するため、部位と治療目的ごとに評価点群や ROI の運用ルールを固定するのが望ましい。

の 3 つである。

## 許容領域シミュレータの設計

### 目的

セットアップマージンは既知であり、評価点の位置もわかっている前提で、`IDENTIFY` の 6DoF 入力に対して、各軸がどこまで許容されるかをシミュレーションする。

特に次を可能にする。

- 現在の 6DoF 条件で `Pass / Fail` を判定する
- 各軸について `あとどこまで動けるか` を数値化する
- 回転 3 軸が遠位点に与える影響を可視化する
- 6 軸スライダーを動かしたとき、他軸の許容範囲が自動で変わる

このツールは単純な判定器ではなく、`許容領域シミュレータ` として設計する。

## 基本コンセプト

判定基準は `ISO から遠い評価点群に対して、実効変位と不確かさ込みの保守的変位がセットアップマージン内に収まるか` である。

6DoF のうち

- `Vertical`
- `Long`
- `Lateral`

は人力でゼロ近傍に合わせやすいことが多い。

一方で

- `Rotation`
- `Pitch`
- `Roll`

は必ずしもゼロにならないため、回転方向の実質的な許容量を可視化することが主な目的となる。

## シミュレータが答えるべき問い

このツールでは、少なくとも次の問いに答えられる必要がある。

### 1. 今の 6DoF でマージン内か

現在値に対して、最悪点がマージン内かを返す。

### 2. 各軸の単独許容量はどこまでか

他の 5 軸をゼロに固定したとき、その 1 軸だけを動かした場合にどこまで許されるか。

### 3. 各軸の条件付き許容量はどこまでか

他の 5 軸を現在値に固定したとき、その 1 軸があとどこまで動けるか。

### 4. どの評価点がボトルネックか

最も先にマージン外に出る点を特定する。

### 5. どの軸が現在の制約因子か

回転か並進か、どの軸が最もマージンを消費しているかを示す。

## 単独許容量と条件付き許容量

この設計では、許容量を必ず 2 種類に分ける。

### 単独許容量

1 軸だけを動かし、他の 5 軸は基準値に固定する。

基準値は通常次のどちらかである。

- `0`
- ユーザーが指定した初期条件

単独許容量は、その軸単体の影響を理解するための指標である。

### 条件付き許容量

他の 5 軸を現在の値に固定したまま、対象軸だけを増減させて許容範囲を求める。

これは実運用上もっとも重要であり、スライダーを動かしたときにリアルタイムで更新されるべき値である。

## 数理モデル

### 入力値

- 軸別セットアップマージン
  - `M_vertical`
  - `M_long`
  - `M_lateral`
- 不確かさパラメータ
  - `U_identify`
  - `U_surrogate`
  - `U_registration`
  - `U_intrafraction`
  - `U_model`
- 安全係数 `z`
- 評価点群 `p_i = (x_i, y_i, z_i)`
- 現在の 6DoF
  - `vertical`
  - `long`
  - `lateral`
  - `rotation`
  - `pitch`
  - `roll`

### 幾何変換

6DoF を

- 並進ベクトル `t`
- 回転行列 `R`

に変換する。

各評価点 `p_i` の変位は

`Δp_i = t + (R - I) p_i`

で計算する。

### 不確かさ合成

総合不確かさは初期設計では RSS で合成する。

`U_total = sqrt(U_identify^2 + U_surrogate^2 + U_registration^2 + U_intrafraction^2 + U_model^2)`

必要に応じて軸別に持つ。

### 保守的変位

各評価点について

`conservative_displacement = effective_displacement + z × U_total`

を定義する。

### 判定条件

軸別判定の基本形は次である。

`abs(Δp_i,k) + z × U_i,k <= M_k`

全評価点、全軸でこれを満たせば `Pass` とする。

## 許容範囲の計算

### 基本方針

各軸について、許容範囲は解析的に決め打ちせず、`数値的に探索` する設計とする。  
理由は次の通り。

- 回転は複数点に非線形に効く
- 複数軸の組み合わせで制約が変わる
- 将来的に DICOM 点群や異方性不確かさを入れても拡張しやすい

### 計算手順

対象軸 `a` の条件付き許容量を求める場合:

1. 現在の 6DoF を基準状態とする
2. 対象軸 `a` 以外の 5 軸を固定する
3. 軸 `a` を正方向に増やし、`Pass` から `Fail` に変わる境界を探す
4. 軸 `a` を負方向に減らし、同様に境界を探す
5. 二分探索で境界値を所定精度まで求める
6. `allowable_min`, `allowable_max` として返す

### 境界探索方法

探索アルゴリズムは次で十分である。

1. 粗い刻みで範囲を広げる
2. `Pass` と `Fail` をまたぐ区間を見つける
3. その区間に対して二分探索する

この方法は堅牢で、閉形式に依存しない。

## 評価点の設計

### 最低限必要な評価点

- `最遠点`
- `遠位点群`

### 推奨

- 最遠点 1 点だけでなく、複数点で評価する
- `ISO からの距離上位 x%` の点を使う
- 部位別テンプレートで点群を保持する

理由は、回転軸との位置関係により、最遠点が必ずしも最悪点にならないためである。

## スライダー連動仕様

このシミュレータの中心仕様は、`1 軸を動かすと他軸の条件付き許容量が自動で更新される` ことである。

そのため、各スライダー変更時に次を再計算する。

1. 現在状態の全点判定
2. 最悪点の更新
3. 6 軸すべての条件付き許容量の再計算
4. 制約因子の更新

## 計算負荷の考え方

評価点数が少ない初期段階では、各スライダー変更時に 6 軸全体の許容量を再計算しても実用上問題ない可能性が高い。  
ただし点群数が増えた場合に備え、次を設計に入れておく。

- 計算関数の純関数化
- キャッシュ利用
- 許容探索の探索幅と精度を設定可能にする

## この設計の利点

- `Pass / Fail` だけでなく、どこまで許容されるかを示せる
- 回転軸の危険性を遠位点ベースで定量化できる
- 不確かさ込みの保守判定ができる
- 部位別プロトコルや点群入力に将来拡張しやすい

## 限界と注意点

- 体表面は真の標的位置の代理であり、完全一致ではない
- surrogate 不確かさを適切に設定しないと過信につながる
- 深部標的や大きな変形のある部位には追加検証が必要
- 回転中心を ISO に固定するモデルが、実運用と完全一致しない場合がある

## 設計上の推奨結論

この機能は、`既存の判定器に追加するオプション` ではなく、`許容領域シミュレータ` として独立した設計概念で扱うのがよい。

中心機能は次の 3 つである。

1. 現在の 6DoF 条件での最悪点判定
2. 各軸の単独許容量
3. 各軸の条件付き許容量

この 3 つを満たせば、セットアップマージンに対して `どの軸がどこまで許されるか` を、数値と視覚の両方で理解できるようになる。
