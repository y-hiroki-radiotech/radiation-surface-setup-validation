# 数理設計書

## 1. 文書目的

本書は、許容領域シミュレータの数理モデルを定義する。  
実装前に、座標系、6DoF の意味、回転の扱い、評価点変位、不確かさ合成、マージン判定、許容量探索に関する数学的前提を固定することを目的とする。

## 2. 対象範囲

本書が対象とするのは次の計算である。

- `IDENTIFY` の 6DoF 入力を剛体変換として扱う
- ISO 基準の評価点に対する変位を計算する
- 不確かさ込みでセットアップマージン判定を行う
- 各 6 軸の単独許容量と条件付き許容量を数値的に求める

本書は、DICOM 読み込み、UI 描画、データ保存仕様は対象としない。

## 3. 前提

### 3.1 幾何モデル

初期実装では、対象は剛体近似できるものとする。  
変形、圧迫、呼吸相変化は直接は扱わず、不確かさ項へ含める。

### 3.2 回転中心

初期実装では、回転中心は `ISO センター` とする。

### 3.3 マージン判定の対象

マージン判定は、標的そのものではなく、`ISO 基準で定義された評価点群` に対して行う。

### 3.4 不確かさの扱い

初期実装では、不確かさ成分は独立とみなし、RSS 合成を採用する。

## 4. 座標系定義

### 4.1 基本座標系

評価点は ISO センターを原点とする直交座標系で表現する。

- `x`: Lateral 軸
- `y`: Long 軸
- `z`: Vertical 軸

各評価点は

`p_i = [x_i, y_i, z_i]^T`

で表す。

### 4.2 単位

- 並進量: `mm`
- 評価点座標: `mm`
- 回転量: 入力は `deg`
- 内部計算の回転量: `rad`

### 4.3 原点

原点は `ISO センター` とする。

## 5. 6DoF 入力定義

入力状態ベクトルを次で定義する。

`s = [v, l, t, r, p, w]^T`

ここで

- `v`: Vertical translation
- `l`: Long translation
- `t`: Lateral translation
- `r`: Rotation
- `p`: Pitch
- `w`: Roll

とする。

注記:

- ここでの `t` は Lateral translation を意味する記号であり、後述の並進ベクトルとは区別する
- 実装では可読性のため、`vertical`, `longitudinal`, `lateral`, `rotation`, `pitch`, `roll` という名称を使う

## 6. 並進ベクトル

入力値から並進ベクトル `T` を次で構成する。

`T = [t, l, v]^T`

ここで順序は `[x, y, z] = [Lateral, Long, Vertical]` に合わせる。

## 7. 回転定義

### 7.1 基本方針

回転は 3 軸回転として扱い、`Rotation`, `Pitch`, `Roll` をそれぞれ基本軸まわりの回転に対応させる。

ただし、実装前に `IDENTIFY` の実機表示と院内運用の符号規約を必ず確認する必要がある。  
本書では、初期設計として次の対応を採用する。

### 7.2 初期対応表

- `Rotation`: Vertical 軸まわり回転
- `Pitch`: Lateral 軸まわり回転
- `Roll`: Long 軸まわり回転

座標系との対応で書くと

- `Rotation`: `z` 軸まわり
- `Pitch`: `x` 軸まわり
- `Roll`: `y` 軸まわり

とする。

注記:

この対応は設計上の初期仮定である。  
実装前レビューで、実機の定義と一致しているかを確認すること。

### 7.3 角度単位変換

入力角度を degree から radian に変換する。

- `θ_r = r × π / 180`
- `θ_p = p × π / 180`
- `θ_w = w × π / 180`

### 7.4 回転行列

各軸回転を次で定義する。

`R_x(θ) = [[1, 0, 0], [0, cosθ, -sinθ], [0, sinθ, cosθ]]`

`R_y(θ) = [[cosθ, 0, sinθ], [0, 1, 0], [-sinθ, 0, cosθ]]`

`R_z(θ) = [[cosθ, -sinθ, 0], [sinθ, cosθ, 0], [0, 0, 1]]`

### 7.5 回転適用順序

初期実装では、合成回転行列を次で定義する。

`R = R_z(θ_r) R_x(θ_p) R_y(θ_w)`

これは

1. Roll
2. Pitch
3. Rotation

の順に局所回転が適用された結果として扱うのではなく、`固定座標系に対して右から順に適用される合成行列` として実装上固定する。

注記:

回転順序が変わると結果も変わるため、この順序はコードとテストで固定し、文書化したまま変更管理する。

## 8. 評価点変位

### 8.1 変位後座標

評価点 `p_i` の変位後座標 `p'_i` は次で与える。

`p'_i = R p_i + T`

### 8.2 変位ベクトル

評価点 `p_i` の変位ベクトル `Δp_i` は次で定義する。

`Δp_i = p'_i - p_i`

よって

`Δp_i = T + (R - I) p_i`

となる。

### 8.3 軸別変位

`Δp_i = [Δx_i, Δy_i, Δz_i]^T`

としたとき

- `Δx_i`: Lateral 方向変位
- `Δy_i`: Long 方向変位
- `Δz_i`: Vertical 方向変位

とする。

## 9. 並進寄与と回転寄与

### 9.1 並進寄与

並進のみの寄与は

`Δp_i^trans = T`

である。

### 9.2 回転寄与

回転のみの寄与は

`Δp_i^rot = (R - I) p_i`

である。

### 9.3 3D ノルム

任意ベクトル `a = [a_x, a_y, a_z]^T` の大きさを

`||a||_2 = sqrt(a_x^2 + a_y^2 + a_z^2)`

とする。

### 9.4 表示用指標

各評価点について次を定義する。

- `translation_only_mm = ||Δp_i^trans||_2`
- `rotation_induced_mm = ||Δp_i^rot||_2`
- `effective_displacement_mm = ||Δp_i||_2`

## 10. 遠位点距離

評価点 `p_i` の ISO からの距離を

`d_i = ||p_i||_2`

で定義する。

遠位点とは、`d_i` が大きい評価点を意味する。

## 11. 小角近似

回転が十分小さい場合、回転に起因する変位は近似的に

`||Δp_i^rot||_2 ≈ ||ω × p_i||_2`

で与えられる。  
ここで角速度ベクトルに相当する小角近似表現を

`ω = [θ_p, θ_w, θ_r]^T`

とする。

さらに、単一回転軸のみを考える場合は

`回転変位 ≈ 距離 × 角度(rad)`

が近似的に成り立つ。

この近似は理解用と sanity check 用に使い、実際の判定では常に回転行列を使う。

## 12. 不確かさモデル

### 12.1 不確かさ成分

少なくとも次を考慮する。

- `U_identify`
- `U_surrogate`
- `U_registration`
- `U_intrafraction`
- `U_model`

### 12.2 初期モデル

初期実装では、各不確かさを `mm` 単位の標準不確かさとして扱う。

### 12.3 RSS 合成

等方性モデルでは、総合不確かさを

`U_total = sqrt(U_identify^2 + U_surrogate^2 + U_registration^2 + U_intrafraction^2 + U_model^2)`

で定義する。

### 12.4 軸別モデル

将来拡張では、各軸 `k` に対して

`U_k = sqrt(U_identify,k^2 + U_surrogate,k^2 + U_registration,k^2 + U_intrafraction,k^2 + U_model,k^2)`

と定義できる。

### 12.5 点別モデル

さらに将来拡張として、点 `i` と軸 `k` の両方に依存する

`U_i,k`

を許容する。

## 13. 安全係数

安全係数を `z` とする。

初期実装では、利用者入力またはプロトコル設定で与える。

代表例:

- `z = 1.0`
- `z = 1.96`
- `z = 2.0`

## 14. 保守的変位

### 14.1 軸別保守的変位

点 `i` の軸 `k` における保守的変位を

`C_i,k = abs(Δp_i,k) + z U_i,k`

で定義する。

### 14.2 3D 表示用保守的変位

表示用として

`C_i^3D = ||Δp_i||_2 + z U_i^3D`

を定義してよい。  
ただし、判定本体は軸別判定を主とする。

## 15. マージンモデル

### 15.1 軸別マージン

セットアップマージンを

- `M_x`
- `M_y`
- `M_z`

で定義する。

対応は次の通り。

- `M_x`: Lateral margin
- `M_y`: Long margin
- `M_z`: Vertical margin

### 15.2 等方性マージン

等方性マージン `M_iso` の場合は

`M_x = M_y = M_z = M_iso`

とする。

## 16. 判定条件

### 16.1 点ごとの軸別判定

評価点 `i` がマージン内である条件を

`C_i,x <= M_x`

`C_i,y <= M_y`

`C_i,z <= M_z`

と定義する。

### 16.2 点ごとの総合判定

点 `i` が `Pass` である条件は

`pass_i = (C_i,x <= M_x) and (C_i,y <= M_y) and (C_i,z <= M_z)`

とする。

### 16.3 全体判定

全評価点について `pass_i = true` であれば全体を `Pass` とする。

## 17. マージン消費量

### 17.1 軸別残余マージン

点 `i` の軸 `k` における残余マージンを

`R_i,k = M_k - C_i,k`

で定義する。

### 17.2 軸別消費率

点 `i` の軸 `k` における消費率を

`Q_i,k = C_i,k / M_k`

で定義する。  
`Q_i,k > 1` は超過を意味する。

### 17.3 ボトルネック軸

全点全軸のうち `Q_i,k` が最大のものを、現在状態における制約因子とする。

## 18. 最悪点

最悪点は、全評価点のうち最大の制約を生む点とする。

初期実装では

`worst_point = argmax_i max(Q_i,x, Q_i,y, Q_i,z)`

で定義する。

## 19. 単独許容量

### 19.1 定義

対象軸 `a` の単独許容量は、他の 5 軸を基準状態に固定したまま、軸 `a` を変化させたときの `Pass` 領域である。

### 19.2 基準状態

初期設計では、単独許容量の基準状態を次の 2 モードで扱えるようにする。

- `zero_based`
- `current_based`

デフォルトは `zero_based` とする。

### 19.3 記法

軸 `a` の値を `u` とし、他軸固定条件のもとでの判定関数を

`F_a(u)`

とする。  
`F_a(u) = true` なら `Pass`、`false` なら `Fail` を意味する。

単独許容量区間を

`A_a^single = [u_a^min, u_a^max]`

と定義する。

## 20. 条件付き許容量

### 20.1 定義

対象軸 `a` の条件付き許容量は、現在の状態ベクトル `s_current` において、軸 `a` 以外を固定したまま軸 `a` を変化させたときの `Pass` 領域である。

### 20.2 記法

条件付き判定関数を

`G_a(u; s_current)`

とする。

条件付き許容量区間を

`A_a^cond(s_current) = [u_a^min(s_current), u_a^max(s_current)]`

と定義する。

## 21. 許容量探索

### 21.1 基本方針

許容量は閉形式でなく、数値探索で求める。

### 21.2 探索レンジ

各軸には探索上限を設定する。

例:

- 並進軸: `[-50 mm, +50 mm]`
- 回転軸: `[-10 deg, +10 deg]`

実装ではプロトコルごとに変更可能とする。

### 21.3 境界探索手順

対象軸 `a` について:

1. 現在値または基準値から開始する
2. 正方向に探索して `Pass` から `Fail` へ変わる区間を見つける
3. 負方向も同様に探索する
4. 各区間に二分探索を適用する
5. 所定誤差以下で停止する

### 21.4 停止条件

二分探索の停止条件は次のいずれかとする。

- 並進軸: 区間幅 `<= 0.01 mm`
- 回転軸: 区間幅 `<= 0.001 deg`
- 反復数上限到達

## 22. 数値安定性

### 22.1 度とラジアン

回転計算直前に必ず degree を radian に変換する。

### 22.2 小角での検証

極小回転では、小角近似の結果と大きく乖離しないことを確認する。

### 22.3 行列計算

回転行列の実装では、数値誤差により `R^T R = I` から僅かにずれることを許容する。

## 23. 初期 sanity check

実装後の確認用に、次の性質が成立することを確認する。

### 23.1 ゼロ入力

全 6 軸が 0 のとき、任意の評価点について

`Δp_i = 0`

となる。

### 23.2 並進のみ

回転が 0 のとき、全評価点について

`Δp_i = T`

となる。

### 23.3 回転のみ

並進が 0 のとき、原点上の点 `p = 0` は不動点となる。

### 23.4 遠位点依存

同一回転角に対して、`||p_j|| > ||p_i||` なら、一般に遠位点の方が大きな回転寄与を受けやすい。

## 24. 未確定事項

実装前に次を確認する必要がある。

- `IDENTIFY` の `Rotation / Pitch / Roll` の正確な軸対応
- 各軸の符号規約
- UI 表示名と内部座標のマッピング
- `Mag` を入力として扱うか、表示専用とするか
- 不確かさの初期値を等方性にするか軸別にするか

## 25. 本書に基づく実装上の結論

初期実装では、`ISO を原点とする評価点群に対して、6DoF を剛体変換として適用し、軸別の保守的変位がセットアップマージンを超えるかどうかで判定する`。  
許容量は、各軸を 1 つずつ変化させたときの `Pass` 境界を二分探索で求める。
